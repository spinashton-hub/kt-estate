## STEP 1. 기준 정의

아래 내용을 복사해 주세요.

### 감사 기준 정의 (Copilot 메모리 저장)
```markdown
- 내 역할은 [IT팀 리더]야. 앞으로 내 시간 사용을 분석할 때는 다음 5개 [핵심 범주]를 기준으로 분류해줘:
[핵심 범주]
(1)전략/기획 (2)고객/대외 (3)조직/인사 (4)내부 운영/보고 (5)개인 업무/집중시간
- 각 범주의 정의를 한 줄씩 정리하고, 애매한 항목은 어디로 분류할지 규칙도 알려줘.
```

## STEP 2. 시간 측정

아래 내용을 복사해 주세요.

### 시간 사용 현황 측정 (기본 감사)
```markdown
- 지난 한 달간의 내 캘린더와 이메일을 검토해서, 내가 가장 많은 시간을 투자한 프로젝트를 5개의 범주로 분류해줘.
- 각 범주별 시간 비율(%)과 간단한 설명을 제공해줘, 표 형태로 정리하고 총 회의 시간과 회의 건수도 함께 보여줘.
```

## STEP 3. 심층 분석

아래 내용을 복사해 주세요.

### 범주별 심층 분석 (Drill-down)
```markdown
- 시간 사용 현황 측정 (기본 감사)에서 비중이 가장 높았던 상위 3개 범주를 각각 더 자세히 분석해줘.
- 각 범주별로 회의 및 이메일 vs 집중작업 비율, 내부 vs 외부 미팅 비율, 반복(정기) 회의 vs 일회성 회의 비율을 나눠서 보여주고, "줄일 수 있어 보이는 시간"이 있으면 근거와 함께 표시해줘.
```

## STEP 4. 진단

아래 내용을 복사해 주세요.

### 시간 배분 진단 (Diagnosis)
```markdown
- 내가 생각하는 이상적인 시간 배분은 전략 40% / 고객 25% / 조직 20% / 내부운영 10% / 개인집중 5% 야.
- 실제 분석 결과와 비교해서 범주별 격차(목표 - 실제)를 표로 보여주고, 가장 격차가 큰 항목 3가지와 그 원인으로 보이는 패턴(예: 특정 정기회의 과다, 보고용 메일 과다)을 짚어줘.
```

## STEP 5. 개선

아래 내용을 복사해 주세요.

### 개선 시나리오 도출 (Recommendation)
```markdown
- 시간 배분 진단 (Diagnosis)에서 진단한 비효율 패턴을 개선할 구체적 방안을 제안해줘.
- 각 방안마다 어떤 회의/활동을 줄이거나 위임 또는 통합할지, 예상 절감 시간(주당 시간), 절감한 시간을 어느 범주로 재배치할지를 포함해서 5개로 정리해줘.
- 실행 난이도(상/중/하)도 표시해줘.
```
