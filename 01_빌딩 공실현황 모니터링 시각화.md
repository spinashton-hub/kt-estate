- 이 파일은 우리 건물의 공간 현황 데이터야, 공실 면적, 공실률, 어느 층이 비어있는지가 핵심이야.
- 전체 내용을 분석해서 층별 공실 현황을 정리해줘위 프롬프트를 md 형식에 맞게 좀 더 구체적으로 다듬어줘

### md(markdown) 결과물 확인 및 복사
```markdown
# 건물 공간 현황 분석 요청

## 1. 파일 개요
- **파일 성격**: 당사 보유 건물의 **공간 현황(Space Inventory) 데이터**
- **핵심 관심 지표**:
  - **공실 면적** (㎡ 또는 평)
  - **공실률** (%)
  - **공실이 발생한 층(Floor) 식별**

## 2. 분석 목적
첨부 파일의 전체 데이터를 검토하여 **층별(Floor-by-Floor) 공실 현황**을 한눈에 파악할 수 있도록 정리한다.

## 3. 분석 요구사항

### 3-1. 데이터 전처리
- 결측치/이상치(예: 면적 0, 음수, 중복 행) 식별 후 처리 방식 명시
- 단위(㎡ ↔ 평) 통일 및 환산 기준 표기

### 3-2. 층별 집계 항목
각 층(B1, 1F, 2F … RF 등)에 대해 다음을 산출:

| 항목 | 설명 |
|------|------|
| 총 임대 가능 면적 | NRA(Net Rentable Area) 합계 |
| 임대 면적 | 현재 계약 중인 면적 합계 |
| **공실 면적** | 총 임대 가능 면적 − 임대 면적 |
| **공실률(%)** | (공실 면적 ÷ 총 임대 가능 면적) × 100, 소수점 1자리 |
| 공실 호실 수 | 비어 있는 호실 개수 |
| 비고 | 전층 공실 / 부분 공실 / 만실 구분 |

### 3-3. 전체 요약(Building-level)
- 건물 전체 **총 공실 면적**, **평균 공실률**
- **공실률 Top 3 층**과 **만실 층** 리스트
- 전층(全層) 공실 또는 90% 이상 공실인 층은 별도 강조 표기

## 4. 산출물(Output) 형식
1. **요약 카드**: 건물 전체 KPI(총 면적 / 공실 면적 / 공실률) 3줄 요약
2. **층별 현황 표**: 위 3-2 항목을 포함한 정렬 가능한 표(공실률 내림차순)
3. **시각화**:
   - 층별 공실률 **막대 그래프**
   - 임대 vs 공실 면적 **누적 막대 그래프**
4. **인사이트 코멘트**:
   - 공실 집중 구간(예: 저층부/고층부) 패턴
   - 임대 전략 관점의 시사점 2~3가지

## 5. 응답 톤 & 스타일
- 전문적이고 간결한 비즈니스 리포트 톤
- 수치는 천 단위 콤마, 공실률은 `%` 단위 명시
- 가정(Assumption)이 필요한 경우 별도 섹션에 명시
```

### 3D 시각화 요구사항 및 주요 지표 구현
```markdown
- 방금 분석한 데이터로 HTML 시각화 파일을 만들어줘.
- 건물을 3D로 표현하되, 마우스로 돌려볼 수 있게 해줘.
- 공실 있는 층은 빨간색으로 반짝이게 강조하고, 그 층에서 선을 뽑아서 몇 층인지 공실 면적이 얼마인지 라벨로 표시해줘.
- 배경은 어두운 색으로, 건물은 유리 사무용 빌딩처럼 창문도 보이게 하고 output 폴더에 저장해줘.
- 층을 클릭하면 오른쪽에서 패널이 열리게 해줘.
- 패널 안에는 그 층의 공간 배치도가 보이고, 공실 공간은 빨간색으로 표시돼서 클릭하면 면적이랑 임차인 정보가 나와야 해.
- 맨 위에는 전체 공실률과 주요 수치 요약을 넣어줘.
- 위 프롬프트를 md파일로 구체화해줘, 이모지 및 아이콘을 모두 제거하고 생성해줘
```

### md(markdown) 결과물 확인 및 복사
```markdown
- 아래 내용으로 html 기반3D 시각화 결과물 제작해줘

# 건물 공실 현황 3D 시각화 + 층별 공간 배치 인터랙티브 HTML 생성 요청서

## 1. 개요
방금 분석한 건물별/층별 공실 데이터를 활용하여, 사용자가 웹 브라우저에서 직관적으로 공실 현황을 파악할 수 있는 단일 HTML 기반 3D 시각화 결과물을 제작한다. 결과물은 유리 외관의 사무용 빌딩을 사실적으로 표현하고, 공실이 발생한 층을 시각적으로 강조하며, 특정 층을 클릭하면 우측에서 해당 층의 공간 배치도 패널이 열려 세부 정보를 확인할 수 있도록 구성한다.

## 2. 목적 (Purpose)
- 건물 단위의 공실 분포를 한눈에 파악할 수 있는 인터랙티브 3D 뷰 제공
- 공실 층의 위치, 면적, 공실률을 라벨로 명확히 전달
- 층 단위 클릭 시 공간 배치도(평면도) 패널로 임차 구획별 상세 정보를 즉시 확인
- 상단 KPI 요약을 통해 임원 보고 및 자산관리(PM/FM) 회의에서 핵심 수치를 즉시 인지

## 3. 입력 데이터 명세 (Input Data Specification)

## 4. 출력 결과물 (Deliverables)
- 저장 경로: `output/building_vacancy_3d.html`
- 형식: 외부 의존성 없이 동작하는 단일 HTML 파일 (CSS, JavaScript 인라인)
- 외부 라이브러리는 CDN 링크로만 로딩

## 5. 기술 스택 (Tech Stack)
- Three.js (r160 이상, CDN) + OrbitControls + CSS2DRenderer
- 평면도 패널은 SVG 또는 HTML5 Canvas 2D로 구현
- HTML5, CSS3, JavaScript ES6+
- 빌드 도구 사용 금지, 순수 정적 HTML 단일 파일로 완결

## 6. 3D 건물 모델링 요구사항
- 건물 형태: 직육면체 코어를 층 단위로 분리한 박스 스택 구조 (층별 개별 Mesh 생성)
- 각 층은 `floor_height_m` 높이의 박스로 모델링하고, Y축 방향으로 순차 적층
- 유리 사무용 빌딩 외관 표현
  - 재질: `THREE.MeshPhysicalMaterial`
  - 주요 파라미터: `transmission`, `roughness`, `metalness`, `clearcoat`, `envMapIntensity` 적용
  - 약간의 색조: 청록 계열 틴팅(예: `#88aabb`)
- 창문 표현
  - 방법 1: 층 외벽 텍스처에 격자 패턴(Canvas로 동적 생성) 매핑
  - 방법 2: `InstancedMesh`로 창문 프레임을 정렬 배치
- 옥상부: 평지붕 + 단순화한 옥탑 구조 1개 배치

## 7. 카메라 및 인터랙션 요구사항
- 컨트롤: `OrbitControls`
  - 좌클릭 드래그: 회전 / 우클릭 드래그: 패닝 / 휠: 줌 / 터치 제스처 지원
- 카메라 초기 위치: 건물 전체가 프레임에 들어오는 아이소메트릭 각도
- 자동 회전(`autoRotate`) 기본 활성화, 사용자 입력 발생 시 즉시 일시 정지
- 토글 버튼으로 자동 회전 ON/OFF 제어

## 8. 공실 층 강조 요구사항
- 공실 판정 기준: `vacant_area_sqm > 0`
- 강조 표현
  - 재질의 `emissive` 색상: 빨강 계열(`#ff2a2a`)
  - `emissiveIntensity`를 시간 기반 사인 함수로 0.3 ~ 1.0 사이에서 펄스(반짝임)
  - 펄스 주기: 약 1.2초
- 완전 공실(`vacant`)과 부분 공실(`partially_vacant`)을 색조 강도로 구분
  - 완전 공실: 최대 강도 1.0, 채도 높음
  - 부분 공실: 최대 강도 0.7, 약간 옅음
- 공실 없는 층: 청백색 은은한 발광(`emissive: #224466`, intensity 0.15)

## 9. 리더 라인(지시선) 및 라벨 요구사항
- 공실 발생 층마다 건물 외부 방향으로 연장되는 선 1개 표시
  - 시작점: 해당 층 외벽 중앙 / 끝점: 건물 폭의 약 1.5배 외측 지점
  - 구현: `THREE.Line` + `LineBasicMaterial`, 색상은 강조 빨강 계열
- 선 끝에 라벨 부착
  - 구현: `CSS2DRenderer` 기반 HTML 라벨
  - 라벨 텍스트(3줄): `N층` / `공실 면적: X m^2` / `공실률: Y%`
  - 라벨 스타일: 반투명 다크 배경, 빨강 테두리, 흰색 텍스트, 모서리 둥글게

## 10. 층 클릭 인터랙션 및 우측 상세 패널 요구사항 (신규)
- 인터랙션 트리거
  - `THREE.Raycaster`로 마우스 클릭 시 어떤 층 Mesh가 선택되었는지 식별
  - 클릭된 층은 외곽선(EdgesGeometry) 또는 outline 강조로 선택 상태 표시
  - 동일 층 재클릭 또는 패널 닫기 버튼 클릭 시 선택 해제
- 우측 슬라이드 패널
  - 위치: 화면 우측에서 좌측으로 슬라이드 인 (`transform: translateX`)
  - 너비: 데스크톱 기준 화면 폭의 30% (최소 360px, 최대 480px)
  - 모바일에서는 전체 폭(100%)으로 표시
  - 애니메이션: 0.3초 ease-in-out
  - 닫기 버튼(우상단 X) 및 ESC 키, 배경 클릭으로 닫기 가능
- 패널 헤더
  - `N층 공간 배치도` 타이틀
  - 부가 정보: 총 임대 면적, 공실 면적, 층 공실률(%)
  - 상태 뱃지: 임대 완료 / 부분 공실 / 완전 공실 (색상으로 구분)
- 패널 본문: 공간 배치도(평면도)
  - 구현: SVG (권장) 또는 HTML5 Canvas
  - 층 평면을 데이터의 `floor_plate_width_m × floor_plate_depth_m` 기준으로 축척 렌더링
  - 각 `unit`을 사각형으로 표시
    - `status == "occupied"`: 회색 계열(`#3a4a6a`), 흰색 테두리
    - `status == "vacant"`: 빨강 채움(`#ff2a2a`), 약간의 반짝임/펄스 또는 호버 시 강조
    - `status == "partially_vacant"`: 빨강과 회색의 사선 패턴 또는 그라데이션
  - 각 unit 사각형 내부에 `unit_id`와 `면적(m^2)` 텍스트 표기
  - 방위 표시(N) 및 축척 바(예: 5m) 우하단 표시
- Unit 클릭 인터랙션
  - 클릭 시 해당 unit이 하이라이트(테두리 노란색 강조)되고 하단 상세 정보 영역에 다음 표시
    - 구획 ID(`unit_id`)
    - 면적(`area_sqm`) m^2
    - 상태(`status`)
    - 임차인(`tenant_name`) 또는 "공실"
    - 임대 종료일(`lease_end_date`) 또는 공실 시작일(`vacant_since`)
  - 동일 unit 재클릭 또는 빈 공간 클릭 시 상세 정보 해제
- 접근성
  - 키보드 Tab 순회 가능, 포커스 표시 명확
  - ARIA 속성(`role="dialog"`, `aria-modal="true"`) 적용

## 11. 상단 KPI 요약 바 요구사항 (신규)
- 위치: 화면 최상단 고정(`position: fixed; top: 0`), 전체 폭, 높이 약 64~80px
- 디자인: 반투명 다크 카드, 좌측 정렬 건물명 + 우측 정렬 KPI 카드 그룹
- 표시 KPI (입력 데이터 `summary` 기반, 누락 시 `floors` 합산으로 자동 계산)
  - 건물명 (좌측, 큰 글씨)
  - 전체 임대 가능 면적 (m^2)
  - 전체 공실 면적 (m^2)
  - 전체 공실률 (%) — 강조 색상으로 표기, 10% 초과 시 빨강 계열
  - 공실 층 수 / 총 층 수
  - 월 임대료 손실 추정액 (KRW, 천 단위 콤마)
- 각 KPI는 `라벨 + 값` 2줄 구성, 값은 큰 글씨(20~24px), 라벨은 작은 글씨(12px)
- 반응형: 모바일에서는 가로 스크롤 가능한 한 줄로 표시

## 12. 배경 및 환경 요구사항
- `scene.background`: 매우 어두운 네이비/블랙 (`#0a0e1a`)
- `THREE.FogExp2` 적용으로 원근 깊이감 부여 (밀도 약 0.015)
- 바닥: `GridHelper` 또는 단순 Plane, 은은한 푸른빛(`#1a2a4a`)
- 조명 구성
  - `AmbientLight`: 형상 인지를 위한 약한 광량
