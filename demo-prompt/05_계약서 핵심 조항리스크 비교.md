## STEP 1. 계약서 업로드

검토 및 분석이 필요한 계약서를 업로드합니다.

## STEP 2. 1차 검토

아래 내용을 복사해 주세요.

### 계약서 검토
```markdown
- 업로드된 계약서를 기반으로 1차 분석을 진행해줘.
 
목표:
- 계약 구조 파악
- 핵심 조항 식별
- 기본 리스크 스캔
 
아래 형식으로 간단하게 정리해줘.
 
[계약 개요]
- 계약 유형 / 목적 / 기간 / 금액 / 지급 구조
 
[핵심 구조 요약]
- 수행 범위 및 산출물
- 일정 및 납기
- 성과 평가 기준(KPI)
- 검수/승인 기준
- 해지/종료 조건
 
[1차 리스크]
- 중요한 리스크 3~5개 (간단 설명 포함)
 
[누락 의심 포인트]
- 2~3개

조건:
- 계약서에 있는 내용 기반으로만 작성
- 조항 위치 가능한 경우 포함
- 애매한 경우 “검토 필요” 표시
```

## STEP 3. 2차 검토

아래 내용을 복사해 주세요.

### 계약서 심층 분석 및 2차 검토
```markdown
- 방금 분석한 내용을 기반으로, 우리에게 불리할 수 있는 조항과 누락된 부분을 더 깊게 분석해줘.
- 특히, 변화관리 프로젝트 기준으로 아래를 중심으로 봐줘.

[불리 조항 기준]
- 결과물 책임 과도
- 평가 기준 모호 또는 불리
- 검수/승인 기준 없음 또는 무제한 요구 가능
- 일정 지연 책임 편향
- 재작업/수정 무제한
- 인력 변경 제한
- 비용 대비 의무 과도
- 성과 미달 시 페널티

출력 형식:

[불리 조항]
- 조항명
- 핵심 내용
- 왜 문제인지
- 실제 발생 가능한 상황
- 위험도 (상/중/하)
- 협상 포인트

[누락/모호]
- 항목
- 문제 이유
- 실제 리스크
- 보완 필요 방향

추가:
- 가장 치명적인 조항 TOP 3를 별도로 강조해서 알려줘.
```

## STEP 4. 결론

아래 내용을 복사해 주세요.

### 최종 결론 정리 및 검토 보고서 작성
```markdown
- 지금까지 분석 내용을 바탕으로 최종 판단을 내려줘.
- 먼저 아래를 간단히 답해줘.

1. 전체 리스크 수준 (낮음 / 보통 / 높음)
2. 수행 난이도 (이유 포함)
3. 비용 대비 리스크 평가
4. 가장 문제 될 3가지

그리고 아래를 수행해줘.

--------------------------------------
[의사결정]
- GO / 조건부 GO / 재검토
- 판단 이유

[반드시 수정해야 할 조항 TOP 5]
--------------------------------------

- 이후, 법무팀 제출용 Word(.docx) 보고서를 생성해줘.

[요구사항]
- 파일 형태로 생성 (다운로드 가능)
- 보고용 문체 (공손하고 간결)
- 구조는 아래를 따를 것

--------------------------------------
제목: 계약 검토 요청 (변화관리 프로그램 계약)

1. 계약 개요  
2. 주요 검토 결과  
3. 주요 리스크 분석  
4. 불리 조항 상세  
5. 누락 및 보완 필요 사항  
6. 사업부 의견  
7. 법무 검토 요청 사항  

[마무리]
본 계약은 사전 검토 결과 일부 리스크가 확인되어 법무 검토 요청드립니다.
--------------------------------------

작성 조건:
- 계약서 근거 기반
- 단정적 표현 금지 ("검토 필요")
- 조항 위치 가능하면 포함 
```
