# 세션 3 — Claude 데모

**One Agenda, Four AI · kt estate 임원 AI 활용 실습교육 · 프롬프트북 v2**

> **사용법**
> - 강사 시연 + 객석 참여(즉석 수정 요청) 형식입니다. 각 코드 블록 우측 상단의 복사 버튼으로 프롬프트를 복사해 사용합니다.
> - `[대괄호]` 부분은 현장에서 실제 산출물로 교체합니다.
>
> **진행 순서:** 맥락 입력(1분) → 장문 분석(7분) → Claude Code 대시보드(12분) → Claude Design 슬라이드(8분) → 브리지(2분)

---

## STEP 1. 공통 맥락 카드 입력

새 Claude 대화를 열고, 아래 내용을 가장 먼저 복사해 주세요.

### 공통 맥락 카드
```markdown
# 프로젝트 맥락
- 회사: kt estate (부동산 개발·운영 전문기업)
- 안건: 서울 ○○동 구 KT지사 유휴부지(연면적 약 5,000평) 복합개발 전략 수립
- 프로젝트 코드명: 리본(Re:Born) — 구 통신 인프라 부지의 재탄생
- 개발 목표: 임대수익 안정성과 자산가치 상승 동시 확보
- 나의 역할: kt estate 개발담당 TF 임원
```

---

## STEP 2. 장문 보고서 통째 분석

부동산 시장 보고서 PDF를 업로드한 뒤, 아래 내용을 복사해 주세요.

### 핵심 인사이트 추출
```markdown
# 역할
- 너는 kt estate 개발 TF의 리서치 분석가다.

# 요청
- 업로드한 시장 보고서에서 시니어 주거·임대시장 관련 내용만 추출하라.
- 임원 보고용 핵심 인사이트 5개로 정리하라.

# 출력 조건
- 각 인사이트에 보고서 내 근거 위치(페이지 또는 섹션)를 표시하라.
- 각 인사이트가 '프로젝트 리본'에 주는 함의를 한 줄씩 덧붙여라.
```

## STEP 3. Claude Code — 인터랙티브 대시보드 생성 ★와우 모먼트

Claude Code(데스크톱 앱 Code 탭 또는 터미널)에서 진행합니다. 아래 순서대로 준비해 주세요.

1. 작업 폴더에 자료를 저장합니다: `컨셉표.md`(세션 1 STEP 6 결과), `시사점.md`(세션 2 STEP 2 결과)
2. 해당 폴더에서 Claude Code를 실행합니다.

아래 내용 복사 / 붙여넣기 이후 진행

자료(ChatGPT컨셉표.md, Gemini시사점.md)

### 대시보드 생성
```markdown
# 요청
- 분석 내용과 함께 위 내용을 종합해 대표이사 보고용 인터랙티브 대시보드를 만들어라.
- 결과는 단일 HTML 파일 'dashboard.html'로 저장하라.

# 포함 요소
- 개발 컨셉 3안 비교 차트
- 시니어 주거 시장 핵심 지표 카드
- 리스크 매트릭스 (발생가능성 × 영향도)

# 디자인
- 제목: '프로젝트 리본(Re:Born) 경영 브리핑'
- KT 브랜드 컬러(레드 포인트)를 적용하라.
- 임원이 클릭하며 탐색할 수 있는 인터랙티브 구성으로 만들어라.

# 완료 후
- 브라우저에서 바로 확인할 수 있도록 파일 경로를 알려줘.
```

> 대화가 아니라 **작업 폴더 안에서 파일을 직접 만들고 고치는 AI** — "말로 시키는 개발"의 순간입니다.

---

## STEP 4. 즉석 수정 요청 (객석 참여)

"어떤 걸 바꿔볼까요?" — 객석 요청을 받아 실시간 반영합니다. 아래는 예시 문형입니다.

### 수정 요청 예시
```markdown
- 차트 색상을 KT 브랜드 컬러(레드 계열)로 통일하고, 리스크 매트릭스에서 '규제 리스크'를 최상단으로 강조해줘
- 핵심 지표 카드에 전년 대비 증감 표시를 추가해줘
```

> Claude Code가 코드를 스스로 수정하고 저장까지 마치면, 브라우저 새로고침으로 변경을 확인합니다.

## STEP 5. Claude Design — 디자인 서식 기반 슬라이드 생성 ★클라이맥스

Claude Design에서 진행합니다. 아래 순서대로 준비해 주세요.

1. Claude Design을 열고 새 작업을 시작합니다.
2. 디자인 서식 파일 **`kt estate.md`**(컬러·서체·레이아웃 규칙)를 업로드합니다.
3. 세션 2 STEP 6에서 복사해 둔 **마크다운 개요**를 준비합니다.

아래 내용을 복사해 주세요.

### 브랜드 슬라이드 생성
```markdown
# 요청
- 첨부한 디자인 서식(kt estate.md)의 규칙을 적용해 임원 보고용 슬라이드를 만들어라.

# 입력
- 디자인 서식: 첨부 파일 'kt estate.md' (컬러, 서체, 레이아웃 규칙)
- 슬라이드 개요: 아래 붙여넣은 마크다운 ('#' 장표 제목, '-' 본문 항목)

# 변환 규칙
- '#' 줄은 장표 제목으로, '-' 줄은 개조식 본문으로 배치하라.
- 디자인 서식의 컬러와 서체 규칙을 모든 장표에 적용하라.
- 장표당 핵심 메시지는 1개로 유지하라.

[여기에 세션 2에서 추출한 마크다운 개요를 붙여넣는다]
```

> 세션 2에서 '이미지라 수정 불가'였던 슬라이드가, **마크다운 바통 + 디자인 서식**으로 브랜드 슬라이드로 다시 태어납니다.

## STEP 6. .pptx 내보내기 — 이미지 슬라이드 구출 완료

아래 순서대로 진행해 주세요.

1. 완성된 슬라이드를 **.pptx로 내보내기 → 다운로드**합니다. 파일명 예: `리본_경영브리핑.pptx`
2. PowerPoint에서 열어 **글자를 직접 수정**해 봅니다 → 이번에는 수정됩니다!

> 직전 세션에서 '안 되던 것'이 되는 순간 — 그것을 가능하게 한 것은 **마크다운 바통**이었습니다.
> 이 .pptx는 **세션 4(Copilot) 실습 파일**입니다 — 보관해 주세요.
> **브리지 멘트:** "도구가 강력할수록, 회사 데이터와 만날 때는 규칙이 필요합니다." → 세션 4(M365 Copilot)로 이어집니다.
